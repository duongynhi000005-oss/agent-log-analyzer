Use Probe as the only broad code-location mechanism.

Rules:
- Preserve task completion quality over token reduction.
- Do not use broad Glob, find, tree, or repository-overview agents.
- Use Probe in small, bounded searches before reading files:
  `npx -y @probelabs/probe@latest search "ParseCSV Owner normalizeName" internal/events --max-results 5 --max-tokens 1200 --format markdown`
  `npx -y @probelabs/probe@latest search "Build OwnerSummary event.Owner event.Service" internal/report --max-results 5 --max-tokens 1600 --format markdown`
  `npx -y @probelabs/probe@latest search "RenderMarkdown Owners table" internal/report --max-results 5 --max-tokens 1200 --format markdown`
- Read only files returned by Probe, using narrow ranges when possible.
- During debugging, rerun targeted package tests first:
  `go test ./internal/events ./internal/report`
- For final verification, run the full suite quietly:
  `go test ./... >/tmp/go-test.log && echo "go test ./... passed" || { grep -A3 -B2 -E "FAIL|--- FAIL" /tmp/go-test.log; exit 1; }`
- Do not use web search.
